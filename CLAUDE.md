# INTEGRAL MVP

제주 물류 경로·공간 공유 플랫폼 - 설득용 시연 MVP

## 📋 프로젝트 개요

**목적**: 투자자/이해관계자에게 3-5분 내 서비스 가치를 전달하는 시연용 프로토타입
**핵심**: 물류 경로와 공간을 지도에서 시각화하고 거래 가능성을 보여주는 웹 애플리케이션
**중요**: 실제 거래 기능 없음. 모든 데이터는 더미 데이터 기반.

---

## 🛠 기술 스택

- **프레임워크**: Vite 5 + React 18 + TypeScript 5
- **스타일링**: Tailwind CSS 3
- **지도** (예정): Kakao Maps API
- **구조**: 싱글 페이지 애플리케이션 (앵커 스크롤 방식)

---

## 📁 프로젝트 구조

```
src/
├── components/
│   ├── layout/
│   │   ├── Navigation.tsx      # 고정 네비게이션 바
│   │   └── Footer.tsx           # 푸터
│   └── common/
│       └── ProductCard.tsx      # 재사용 가능한 상품 카드
│
├── types/
│   └── models.ts                # 모든 타입 정의
│
├── App.tsx                      # 메인 싱글 페이지 (섹션별 플레이스홀더)
└── main.tsx
```

**추후 추가 예정**:
- `src/data/mockData.ts` - 더미 데이터
- `src/components/hero/` - 지도 히어로 섹션
- `src/components/deal/` - 거래 모달
- `src/utils/` - 규정 엔진, 비용 계산기

## 🎯 핵심 설계 원칙

1. **직관성 우선**: 별도 설명 없이 화면만으로 서비스 이해 가능
2. **지도 중심 UI**: 모든 가치는 지도에서 시각적으로 전달
3. **싱글 페이지 구조**: 페이지 이동 없이 스크롤/모달로 모든 정보 접근
4. **더미 데이터 기반**: 백엔드 없음, 프론트엔드에서 모든 로직 처리
5. **룰 기반 로직**: 복잡한 알고리즘 대신 간단한 if-else 조건문

## 🎬 데모 시나리오 (3-5분)

1. **랜딩** (10초): 지도에서 경로/공간 즉시 인식
2. **지도 탐색** (30초): 확대/축소, 경로/공간 클릭
3. **상품 탐색** (1분): 카드 리스트 → 상세 모달
4. **거래 프로세스** (2분): 화물 조건 입력 → 규정 매칭 → 예약 성공
5. **차별화 기능** (1분): 경로 통합, 규정 자동 매칭
6. **마무리** (30초): 서비스 가치 요약
